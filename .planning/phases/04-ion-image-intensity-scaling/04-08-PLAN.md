<objective>
Fix three non-rasterize defects from the Phase 4 code review:

1. **WR-01 (store.ts):** `renderIonImage` sets `mzWindow: {mz, tolDa}` even when `extractXIC` returns null, causing the SPEC-02 amber band to appear on the spectrum with no ion image visible — inconsistent UI state. Fix: set `mzWindow` only when `ionImage` is non-null.

2. **WR-03 (ImagingPanel.tsx):** `ImagingPanel` subscribes to `mzWindow` from the store but never uses the value (guarded by `void mzWindow`). This causes a redundant re-render on every m/z query. The `ionImage` subscription already covers the re-render. Fix: remove the dead subscription.

3. **REQUIREMENTS.md traceability:** IMAGE-02 and IMAGE-03 are marked `[ ]` (Pending) despite being implemented and verified. Fix: update to `[x]` and change status to "Complete".

Purpose: Eliminate the inconsistent amber-band-without-ion-image UI state and remove a redundant re-render per m/z query.
Output: Corrected store.ts + cleaned ImagingPanel.tsx + updated REQUIREMENTS.md.
</objective>
