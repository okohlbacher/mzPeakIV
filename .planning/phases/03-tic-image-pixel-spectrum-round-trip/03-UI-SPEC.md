# Phase 3 — UI Design Contract

> Visual and interaction contract for the TIC image + pixel→spectrum round-trip. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Mode: MVP.** This phase is explicitly minimal/functional/correct — no design system, no polish (deferred to Phase 5). All values below codify the *existing* inline-style conventions of the two shipped phases (App.tsx, SpectrumPanel.tsx, GridDiagnosticsPanel.tsx) plus the four phase-specific visual contracts the imaging panel introduces: the TIC **colormap**, the **sparse-pixel sentinel**, the **selection ring**, and the **hover readout** format. The dominant visual artifact of this phase is a data-driven Canvas heatmap, not chrome — most "color" lives in the scientific colormap, which is a correctness surface (perceptual uniformity), not a brand palette.

## Spacing Scale

The codebase expresses spacing in `rem` (root font ≈ 16px) and a few fixed px layout constants. Mapped to the 4px grid for the checker; **executors must reuse the existing `rem` literals already present in sibling panels**, not introduce new ad-hoc values.

| Token | Value | rem equivalent | Usage in this phase |
|-------|-------|----------------|---------------------|
| xs | 4px | ~0.25rem | Hairline gaps (rarely needed) |
| sm | 8px | 0.5rem | Panel padding (`padding: "0.5rem"`), section gaps below canvas, label↔control gaps |
| md | ~12px | 0.75rem | Header padding, inline separators (`marginLeft: "0.75rem"`) |
| lg | 16px | 1rem | Heading bottom margins |
| xl | 32px | 2rem | (not used this phase) |

Fixed layout constants (inherited, do not change): left inspection column `width: 400` px; `1px solid #ddd` / `#eee` panel separators; canvas selection ring `lineWidth: 1`.

Exceptions:
- **Canvas intrinsic pixel grid** is sized `grid.width × grid.height` device px and CSS-scaled for display — this is a data dimension, NOT a spacing token. It is exempt from the 4px scale.
- **Pixel aspect ratio** (`IMS:1000046/47`, `grid.pixelSizeUm`) governs the canvas CSS `width:height` and is a correctness constraint, not spacing. Do not round it to the 4px grid.
- Canvas max display width capped to the right pane; preserve aspect (`object-fit`-style scaling via CSS width/height or `aspect-ratio`), never stretch to square.

## Typography

Inherited from App.tsx root (`system-ui, sans-serif`). No custom font sizes/weights beyond what sibling panels already use. Two weights only: regular (400, browser default) and the table-header semibold (600) already in GridDiagnosticsPanel.

| Role | Size | Weight | Line Height | Notes |
|------|------|--------|-------------|-------|
| Panel heading | `<h2>` browser default (~1.5rem / 24px) | 600 (browser default for h2) | 1.2 | "TIC Image" / "Spectrum" section headings — matches existing `<h2>` in SpectrumPanel |

## Color

Two-tier palette: (1) **chrome colors** — inherited verbatim from the existing panels, applied to borders, text, and the one interactive accent; (2) the **TIC colormap** — a data-encoding LUT that occupies most of the visible surface but is NOT part of the 60/30/10 chrome budget (it encodes intensity, not hierarchy). The 60/30/10 split below describes the *chrome* surrounding the canvas.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` | App background, panel surfaces (no explicit bg set; browser white) |
| Secondary (30%) | `#ddd` / `#eee` borders, `#555` / `#888` muted text | Panel separators (`1px solid #ddd` header, `1px solid #eee` between panels), secondary/caption text |
| Accent (10%) | `#1565c0` (blue) | **Reserved list below** |
| Warning (semantic) | `#8a6d00` (amber) | Caution only — mixed profile/centroid representation warning (D-08); reuses the existing `WARNING` constant from GridDiagnosticsPanel. NOT an error. |
| Destructive | not applicable | Phase 3 has no destructive actions |

**Accent (`#1565c0`) reserved for:**
- The uPlot spectrum line stroke (already established in SpectrumPanel).
- (Optional) the selected-pixel emphasis in the hover/heading text if a single accent cue is wanted.
- Nothing else. Do not accent every interactive element; the canvas itself is the primary interactive surface and signals interactivity via the `cursor` and the hover readout, not color.

### Canvas data colors (correctness surface — NOT chrome)

| Element | Value | Rule |
|---------|-------|------|
| TIC colormap | **Viridis-like perceptually-uniform LUT** (default) — a single fixed colormap; gray-ramp acceptable fallback | Recommended over a raw gray ramp: perceptually uniform, colorblind-safe, the MSI community standard. Implemented as one pure function `colormap(norm ∈ [0,1]) → [r,g,b]`. **No selector** — colormap choice UI is Phase 4 (IMAGE-03). Keep it a single swappable function so Phase 4 can add the selector without refactor. |
| Sparse / absent pixel sentinel | `#1a1a1a` (near-black, RGBA `26,26,26,255`) | **D-09 / C8 — MANDATORY distinction.** `presenceMask[k] === 0` (no spectrum acquired) renders as this sentinel, NEVER as colormap-bottom. Absent ≠ zero-intensity. The sentinel must be visually distinct from *every* value the colormap can produce (viridis bottom is dark purple `~#440154`, so `#1a1a1a` near-black reads as clearly different). If the chosen colormap's low end approaches near-black, switch the sentinel to a neutral mid-gray (`#444`) — the invariant is *distinguishable*, not the exact hex. |
| Zero-intensity pixel | colormap value at `norm = 0` (e.g. viridis dark purple) | Real signal of value 0 — uses the colormap bottom, distinct from the sentinel above. |
| Selection ring | 1px stroke; `#ffffff` (white) default, or `#000000` (black) if the underlying pixel is light | **D-06.** Drawn as a 2nd-pass `strokeRect(x0 + .5, y0 + .5, 1, 1)` *after* `putImageData`. Contrast-pick white vs black from the selected pixel's luminance so the ring is always visible against both colormap extremes and the sentinel. `lineWidth: 1`. |

> Contrast/accessibility note: viridis is chosen specifically because it is perceptually uniform and deuteranopia/protanopia-safe — for a scientific heatmap this is the accessibility contract, more so than chrome WCAG ratios. The hover readout text (`#000` on white at 0.8rem) and warning amber (`#8a6d00` on white) both clear WCAG AA.

## Copywriting Contract

Imaging mode is the new surface. Copy is terse, scientific, lowercase-keyed where it matches existing panels (e.g. `id:`, `px filled`). 1-based coordinates in all user-facing readouts (C3).

| Element | Copy |
|---------|------|
| Primary CTA | **No button CTA.** The primary action is *click a pixel* on the canvas — the canvas itself is the affordance (`cursor: crosshair` or `pointer`). No "Select pixel" button; the interaction is direct manipulation. |
| TIC panel heading | `TIC Image` (or `Total Ion Current`) |
| Hover readout (pixel present) | `x: {x1}, y: {y1} · TIC: {value}` — `x1`/`y1` are 1-based (`x0 + grid.coordinateBase`); `value` formatted compact (e.g. `1.4e6`). Matches the `·`-separated style of SpectrumPanel's `id: … · N points`. |
| Hover readout (pixel absent) | `x: {x1}, y: {y1} — no data` — shown when `presenceMask[key] === 0`; muted color (`#888`). |
| Hover readout (mouse left canvas) | empty string (label disappears — D-05) |
| Selected-pixel spectrum heading | SpectrumPanel heading reads `Spectrum — pixel ({x1}, {y1})` when a pixel selection drove the load (imaging mode); falls back to plain `Spectrum` for index-driven / non-imaging selection. |
| Empty state heading | `No imaging grid` |
| Empty state body | For a valid **non-imaging** file: reuse the existing GridDiagnosticsPanel notice `Not imaging data — no spatial coordinates found` — the TIC panel is simply **not rendered** (App.tsx renders `ImagingPanel` only when `grid !== null`). Do NOT show an empty canvas or an error for non-imaging files. |
| TIC-unavailable body | If grid exists but `tic === null` (extractXIC returned nothing): `Could not compute TIC for this file` (muted) — distinct from an error banner. |
| Mixed-representation warning | `Mixed profile/centroid spectra — TIC computed from {majority} ({n} of {total}); per-pixel spectra route individually` — amber `#8a6d00`, in the diagnostics area, collapsible per the existing pattern (D-08). |
| Error state | Real load/read failures continue to flow through the existing `ErrorBanner` (`StoreError { class, message }`), unchanged. Centroid route with no rows → named throw → ErrorBanner (Pitfall 7); never a silent blank spectrum. |
| Destructive confirmation | not applicable — no destructive actions in this phase |

## Interaction Contract (phase-specific)

Captured here for the executor as the interaction source of truth (beyond the 6 checker dimensions).

| Interaction | Contract | Source |
|-------------|----------|--------|
| Hover canvas | Compute `(x0,y0)` via bounding-rect math (resolution/aspect-safe — NOT `offsetX`); update the readout label; show `— no data` over absent pixels. Label clears on `mouseleave`. | D-05, Pitfall 5 |
| Click canvas | Same hit-test → `key = y0*grid.width + x0` → guard `presenceMask[key] !== 0` → `grid.coordToSpectrumIndex.get(key)` → `selectSpectrum(idx)`. Clicking an absent pixel is a no-op (no selection change). | D-04 |
| Selection feedback | 1px contrast ring repainted on `selectedIndex` change (2nd canvas pass). The clicked pixel's spectrum appears in the existing SpectrumPanel in-place. | D-06 |
| Numeric index input | Retained as secondary UX in SpectrumPanel (unchanged) — still drives `selectSpectrum`; useful for non-imaging files and debugging. The two selection paths share one store action. | D-04 |
| Orientation | **Fixed**: `M[row][col]`, col=x, row=y, (1,1) top-left, y-down, NO flip/transpose. The grid key `y0*width+x0` already encodes this — any flip in the rasterizer is a correctness bug. | C2, IMAGE-04 |
| Pixel aspect | Canvas CSS dimensions honor `grid.pixelSizeUm` (x:y); never force square when aspect ≠ 1:1. | C5, IMAGE-04 |
| Cursor | `crosshair` (or `pointer`) over the canvas to signal the pixel-pick affordance. | this spec |

## Layout Contract

`App.tsx` `stage === "ready"` block, right of the 400px inspection column:

- **Imaging file (`grid !== null`):** right pane stacks **TIC `ImagingPanel` (top) + SpectrumPanel (below)**, vertically. Canvas sized to fit the pane width, height by aspect; hover readout label sits directly below the canvas; spectrum chart below that.
- **Non-imaging file (`grid === null`):** right pane is the existing **SpectrumPanel only**, unchanged. No empty canvas, no placeholder.

This is functional plumbing only; refined layout/responsiveness is Phase 5 scope.
