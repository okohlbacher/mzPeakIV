# Phase 4 — UI Design Contract

> Visual and interaction contract for the Ion Image + Intensity Scaling phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope note:** mzPeakIV is a *format-orientation tool* for wet-lab scientists / format implementers, not a consumer product. It uses **inline React styles** (no Tailwind, no CSS framework, no shadcn) — verified by codebase scan: no `components.json`, no `tailwind.config.*`, no `.css`/`.scss` files in `src/`. This contract therefore **documents the established inline-style system already shipped in Phases 1–3** and the phase-specific additions, rather than introducing a new design system. CONTEXT D-07 explicitly scopes Phase 4 to "functional plumbing only; visual polish is Phase 5." The contract is intentionally minimal and matches the existing utilitarian aesthetic.

## Spacing Scale

The existing codebase expresses spacing in `rem` (root font-size 16px ⇒ `0.5rem` = 8px). Declared scale, normalized to 4px multiples and mapped to the rem values already in use (`0.5rem` and `0.75rem` dominate — verified: 14× `0.5rem`, 21× `0.75rem`):

| Token | Value | rem equivalent | Usage |
|-------|-------|----------------|-------|
| xs | 4px | 0.25rem | Inline gaps between control-row items (label ↔ input) |
| sm | 8px | 0.5rem | Default element/section padding, gap between stacked rows (the dominant existing unit) |
| md | 12px | 0.75rem | Header padding, inter-control horizontal gap in the controls row (matches `App.tsx` header) |
| lg | 16px | 1rem | Larger separation if a control row wraps |

Exceptions:
- **Canvas dimensions are data-driven, not token-driven.** The ion-image `<canvas>` intrinsic size is `grid.width × grid.height` device pixels (one device pixel per grid cell); its CSS display size is `width: 100%` capped by the pane with `aspectRatio` from `IMS:1000046/47` pixel size (C5). This is a pixel-exact rendering surface and is intentionally exempt from the spacing scale — identical to the Phase 3 TIC canvas (`ImagingPanel.tsx:131-133`).
- **uPlot internal chart geometry** (axes, gutters, the SPEC-02 band) is owned by uPlot's own layout and the `hooks.draw` device-pixel math (`valToPos(..., 'x', true)`), not this scale.

## Typography

The app sets `system-ui, sans-serif` once at the root; sizes are expressed in `rem`. Existing UI uses exactly: default body (`1rem`/16px, browser default), `0.8rem` (the pervasive small/secondary size — 9 occurrences), and `<h2>` headings (browser default ~`1.5rem`). Two weights only: regular (400, default) and the bold of `<h1>`/`<h2>` headings (700, browser default). Declared:

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / control input | 16px (1rem) | 400 | 1.5 |
| Label / control caption | 16px (1rem) | 400 | 1.5 |
| Secondary / stats readout / hover readout / warning | 12.8px (0.8rem) | 400 | 1.4 |
| Heading (panel `<h2>`, e.g. "Ion Image") | ~18px (browser `<h2>` in this context) | 700 | 1.2 |

Notes for the executor:
- The Phase-4 **stats readout line** (D-11) and the **ion-image hover readout** (D-06) MUST use `fontSize: "0.8rem"` to match the existing TIC hover readout (`ImagingPanel.tsx:219`) and the mixed-representation warning (`ImagingPanel.tsx:185`).
- The new **"Ion Image" panel heading** MUST be an `<h2>` with `margin: "0 0 0.5rem"` to match the existing "TIC Image" heading (`ImagingPanel.tsx:180`).
- Do not introduce additional font sizes or weights. Three sizes (16 / 12.8 / heading) and two weights (400 / 700) total.

## Color

The established palette is a light "lab-tool" surface set plus a single blue interactive accent, three semantic colors, and the data-domain heatmap colormaps (which are *data encoding*, not chrome). Verified token frequencies: `#888` muted (7×), `#1565c0` accent (6×), `#ddd`/`#eee` borders (8× combined).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` / `#fafafa` | Page background, canvas-section background, panel surfaces |
| Secondary (30%) | `#f5f5f5` / `#eee` / `#ddd` | Borders (`1px solid #ddd`), dividers, the left inspection column rule, control hairlines |
| Accent (10%) | `#1565c0` | See reserved-for list below |
| Muted text | `#888` (also `#555`/`#666` for slightly-darker secondary) | Disabled/secondary readout text, "no data" hover state |
| Destructive / error | `#b71c1c` text on `#fdecea` bg, `#f5c6cb` border | Error banner only (existing `ErrorBanner.tsx`) — Phase 4 raises **no new destructive surface** |
| Caution / warning | `#8a6d00` / `#664d03` text, `#ffc107` border, `#fff3cd` bg | Non-error cautions (e.g. the mixed-representation warning reused from Phase 3 D-08) |

**Accent (`#1565c0`) reserved for:**
- The **"Show Ion Image"** primary button (the one explicit action in the controls row, D-01).
- The **uPlot spectrum series stroke** (existing, `SpectrumPanel.tsx:78` — unchanged).
- Focus/active state of the controls-row inputs (browser-native focus ring is acceptable; if styled, use `#1565c0`).
- Accent is **NOT** used for: every input border, the colormap/scale/percentile selects (these are neutral `#ddd`-bordered native controls), the canvas, or the stats line.

**Domain colormaps (data encoding — separate from UI chrome, IMAGE-03 / D-08):**

| Colormap | Anchors / rule | Notes |
|----------|---------------|-------|
| Viridis (default) | existing 9-stop matplotlib viridis in `rasterize.ts` (bottom ≈ `#440154`) | Already shipped; the default for both TIC and ion image. |
| Inferno | matplotlib 9-stop inferno LUT (`[0,0,4] … [252,255,164]`) — `[ASSUMED]`, cosmetic only | Add alongside `viridis()`; snapshot-test monotonic luminance, NOT exact RGB (RESEARCH A1). |
| Grayscale | `t → [round(t·255), round(t·255), round(t·255)]` | Trivial; useful for print/contrast checks. |
| Absent-pixel sentinel | `#1a1a1a` (`[0x1a,0x1a,0x1a]`, existing `SENTINEL`) | MANDATORY (C8 / D-06): absent pixels render distinctly from a genuine in-window-zero present pixel. Applies to the ion-image rasterizer identically to TIC. |

**SPEC-02 m/z-window band (D-12):** translucent amber `rgba(255,200,0,0.25)` filled rectangle over `[mz−tolDa, mz+tolDa]` on the uPlot spectrum. `[ASSUMED]` cosmetic (RESEARCH A2) — chosen to read against the `#1565c0` series stroke; the executor may adjust opacity/hue if it clashes, but it MUST remain a low-alpha fill (never an opaque overlay that hides the trace) and MUST be amber-family (distinct from the blue series and from any error/warning semantic).

## Copywriting Contract

All copy is React-escaped string interpolation (no `innerHTML`). Numeric values format via the existing `formatCompact()` helper (`ImagingPanel.tsx:35`).

| Element | Copy |
|---------|------|
| Primary CTA | **Show Ion Image** (button label; D-01/D-07) |
| Control labels | `m/z`, `±`, `tolerance`, unit select shows `Da` / `ppm`, colormap select, scale select shows `linear` / `log`, percentile select shows `90th` / `95th` / `99th` / `99.9th` (D-07/D-08/D-09/D-10) |
| Ion-image hover readout (present pixel) | `x: {N}, y: {N} · intensity: {formatCompact(value)}` — 1-based coords; "intensity" = XIC-aggregated value, NOT "TIC" (D-06) |
| Ion-image hover readout (absent pixel) | `x: {N}, y: {N} — no data` (muted `#888`; mirrors existing TIC "no data" readout, `ImagingPanel.tsx:147`) |
| Stats / blank-prevention line (D-11, SC-3) | `{N} / {total} pixels with signal · range {min}–{max} · scale: {linear\|log} ({Nth} pct)` — always visible once the ion image is rendered; values via `formatCompact()` |
| Empty state (before first render, D-04) | **No ion-image canvas and no placeholder is rendered until the first "Show Ion Image" click.** The controls row is the only visible Phase-4 surface beforehand. (Verified intent: D-04 — "The ion image canvas is not rendered (no placeholder) until first render.") |
| Zero-signal result (image rendered, all present cells in-window-zero) | The stats line itself is the empty-state message: `0 / {total} pixels with signal · range 0–0 · scale: {mode} ({Nth} pct)` — this is the explicit "blank because absent vs blank because scaling" disambiguation SC-3 mandates. No separate banner. |
| Mixed-representation caution (reused) | `⚠ {mixedRepresentationWarning}` in caution amber `#8a6d00`, `0.8rem` — reuse the Phase-3 surface verbatim (D-08 routing) |
| Error state | Reuse the existing `ErrorBanner` via `store.classifyError` (no new error UI in Phase 4 — full taxonomy is Phase 5/UX-01). A failed `extractXIC` sets `stage: "error"` and the existing banner renders the named/actionable message. |
| Destructive confirmation | **None.** Phase 4 has no destructive action — entering an m/z, rendering, recoloring, and clicking a pixel are all non-destructive and freely reversible. |

**Input validation copy (V5, RESEARCH Security):** the "Show Ion Image" handler rejects non-finite / non-positive `m/z` or `tolerance` (and guards `mz − tolDa < 0`) **before** calling `extractXIC`. On invalid input the button is a no-op (disabled or silently rejecting) — no destructive consequence, consistent with the existing `SpectrumPanel` index-input guard (`SpectrumPanel.tsx:137-140`). If a visible hint is added, keep it `0.8rem` muted; a disabled-button affordance is sufficient and preferred for this tool.

## Interaction Contract (phase-specific)

Documented because the executor and checker need the interaction states explicitly (these are the load-bearing behaviors, distilled from CONTEXT D-01..D-12):

| Interaction | Behavior | Source |
|-------------|----------|--------|
| Enter m/z + tolerance + unit, click **Show Ion Image** | File read via `extractXIC(null, {start: mz−tolDa, end: mz+tolDa}, useProfile)`; caches `ionImage: Float32Array`; renders the ion-image canvas below the controls row. The **only** path that touches the file. | D-01 / D-04 |
| Change colormap / scale / percentile | **Re-rasterizes the cached `Float32Array` only** — NO `extractXIC`, NO file I/O, NO spinner. Auto-applies on change (no button). | D-02 / SC-5 (Pitfall 2) |
| Hover the ion-image canvas | Readout updates with 1-based `x, y · intensity`; clears on mouse-leave; absent pixels show "— no data" muted. | D-06 |
| Click an ion-image pixel | Calls `selectSpectrum(idx)` (same as TIC canvas); clicking an absent pixel is a no-op. Selection ring drawn with luminance-contrast color (reuse `ImagingPanel` ring pass). | D-05 / D-06 |
| Pixel selected (either canvas) | `mzWindow` band appears on the spectrum if an m/z has been entered (`mzWindow !== null`). | D-05 / D-12 |
| Pan/zoom the spectrum | The SPEC-02 band recomputes every redraw inside `hooks.draw` via `valToPos(..., 'x', true)` — never frozen/stale. | D-12 (Pitfall 5) |

**Layout (D-04/D-07):** controls row sits **above** the TIC canvas; the TIC canvas remains visible; the ion-image canvas appears **below** the controls (conditionally, post-first-click). Two separate canvases, shared orientation by construction (no flip/transpose — C2 MANDATORY for the ion-image rasterizer too).
