# Phase 2 ‚Äî UI Design Contract

> Visual and interaction contract for the Grid Diagnostics panel. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope is intentionally MINIMAL.** This phase adds ONE new UI surface ‚Äî `GridDiagnosticsPanel` ‚Äî an inline expandable section appended to the existing left inspection column below `CapabilitiesPanel`. It MUST reuse the established inline-style panel pattern. Do not introduce a design-system dependency, CSS framework, or new visual language. All tokens below are extracted from the existing Phase 1 panels (`CapabilitiesPanel.tsx`, `StatsPanel.tsx`) and are binding.

## Spacing Scale

The existing panels use `rem`-based inline values. To keep this phase's panel pixel-identical to its siblings, the contract is expressed in the **exact `rem` tokens already in the codebase**, with their 16px-base px equivalents shown for the 4-point-grid check.

| Token | Value | px (16px base) | Usage (binding ‚Äî copied from CapabilitiesPanel/StatsPanel) |
|-------|-------|----------------|-------------|
| section-padding | `0.5rem` | 8px | `<section>` padding (the panel's outer padding) |
| label-gap | `0.4rem` | ~6.4px | `<h3>` bottom margin (`margin: "0 0 0.4rem"`) ‚Äî matches sibling headings exactly |
| cell-gap | `0.75rem` | 12px | `paddingRight` on row label `<th>` (column gutter) |
| list-indent | `1rem` | 16px | `<ul>` left padding for nested lists (disagreements / anomaly lists) |

Exceptions: The `0.4rem` heading margin is non-4-grid but is an **intentional exact match** to the existing `CapabilitiesPanel`/`StatsPanel` `<h3>` styling ‚Äî visual consistency with sibling panels overrides the 4-point grid here. Do not "fix" it to `0.5rem`; that would make this panel's heading misalign with the three panels above it. No other exceptions.

## Color

Palette extracted verbatim from the existing panels. The new panel uses ONLY these existing values ‚Äî it introduces no new colors.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` (page/panel background ‚Äî inherited, never set explicitly) | Panel surface background |
| Secondary (30%) | `#eee` / `#ddd` (border greys) | `borderTop: "1px solid #eee"` panel separator (matches StatsPanel/CapabilitiesPanel); `#ddd` is the column/header rule used by App/header |
| Accent (10%) | `#1b5e20` (success green, from CapabilitiesPanel "imaging detected: yes") | Reserved ONLY for the all-clear / healthy-grid affirmation when reconstruction is complete and clean (filled == total, no duplicates). Used sparingly. |
| Muted text | `#888` | The non-imaging notice text, "1:1 (assumed)" aspect note, and any "‚Äî" placeholder values. This is the established muted/secondary text grey. |
| Warning | `#8a6d00` (amber-brown, accessible on white; NOT in Phase 1 palette) | The single new semantic color: anomaly indicator (`‚öÝ` glyph + warning text) when `filledCount < totalCells` (sparse), duplicates > 0, or a discovery-vs-authoritative disagreement exists. This is a CAUTION state, NOT an error. |

Accent reserved for: the healthy-grid green affirmation only (e.g. a `‚úì`-style "100% filled" emphasis). Never used for generic text, borders, or links.

**Critical color rule ‚Äî non-error states:** The non-imaging notice (D-04) and the anomaly/warning state (sparse grid, duplicates, disagreement) MUST NOT use the error red `#b71c1c`. Red is reserved by Phase 1 exclusively for `unsupported` findings and hard errors. A sparse or non-imaging grid is a **valid, expected outcome**, not a failure.

- Non-imaging notice ‚Üí muted grey `#888`, informational tone.
- Grid anomaly (sparse / duplicate / disagreement) ‚Üí warning amber `#8a6d00` + `‚öÝ`, caution tone.
- Healthy grid ‚Üí neutral `#888`-or-default text, optionally with `#1b5e20` accent on the "filled" figure.

Destructive: none ‚Äî this phase has no destructive actions.

## Copywriting Contract

All copy is locked by CONTEXT.md (D-03, D-04) and the `<specifics>` block. Values are exact strings, not paraphrases.

| Element | Copy |
|---------|------|
| Panel heading | `Grid` |
| Compact summary line (healthy) | `Grid: {width}√ó{height} ‚Äî {filledCount}/{totalCells} px filled ({pct}%)` ‚Äî e.g. `Grid: 260√ó134 ‚Äî 34840/34840 px filled (100%)`. Use `√ó` (U+00D7), an em-dash-spaced separator, and `toLocaleString()` for thousands grouping on the counts. |
| Compact summary line (anomaly) | Same line, prefixed with `‚öÝ ` and rendered in warning amber. Anomaly = `filledCount < totalCells * 0.95` OR `duplicateCount > 0` OR a discovery-vs-authoritative disagreement. |
| Expand/collapse affordance | Click the compact line to toggle. Provide an accessible disclosure: a leading triangle glyph `‚ñ∏` (collapsed) / `‚ñæ` (expanded), `role`/`aria-expanded` per the existing collapsible pattern, and `cursor: "pointer"` on the clickable row. |
| Expanded ‚Äî Dimensions row | label `Dimensions` ‚Üí value `{width} √ó {height} px` |
| Expanded ‚Äî Fill row | label `Fill` ‚Üí value `{filledCount} / {totalCells} pixels ({pct}% filled)` |
| Expanded ‚Äî Spectra row | label `Spectra` ‚Üí value `{uniqueCoordCount} unique coords / {spectrumCount} spectra` |
| Expanded ‚Äî Missing row | label `Missing` ‚Üí value `{missingCount} pixels` (omit row or show `0` when none) |
| Expanded ‚Äî Duplicate row | label `Duplicates` ‚Üí value `{duplicateCount} pixels` (warning amber when > 0) |
| Expanded ‚Äî Pixel size row | label `Pixel size` ‚Üí value `{x} √ó {y} ¬µm` or `1:1 (assumed ‚Äî no IMS:1000046/47)` in muted grey when absent |
| Expanded ‚Äî Coord source row | label `Coord source` ‚Üí value the winning CoordSource strategy label, e.g. `promoted scan columns`, `scan.parameters cvParams`, or `spectrum id parse` |
| Expanded ‚Äî Disagreement note | label `Discovery check` ‚Üí value `Declared {discoveryDims} vs observed {observedDims} ‚Äî using declared` in warning amber when they disagree; `agrees` (muted) when consistent |
| Non-imaging notice (D-04) | `Not imaging data ‚Äî no spatial coordinates found` ‚Äî single line, muted grey `#888`, shown in place of the compact summary line. No dimensions, no expand affordance, no diagnostics table. |
| Empty/loading state | Panel renders nothing (`return null`) until `stage === "ready"`, identical to `StatsPanel`/`CapabilitiesPanel` (`if (!stats) return null`). No skeleton needed ‚Äî grid build is eager and fast (D-05). |
| Error state | None owned by this panel. Hard load failures are surfaced by the existing `ErrorBanner` at the App level; a non-imaging file is NOT routed there (it is the muted notice above). |

Primary CTA: none ‚Äî this is a read-only diagnostics surface with a single disclosure (expand/collapse) interaction, no primary action button.

Destructive confirmation: not applicable ‚Äî no destructive actions in this phase.
