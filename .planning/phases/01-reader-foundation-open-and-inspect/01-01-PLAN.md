<objective>
Walking Skeleton for mzPeakExplorer: scaffold a Vite 8 + React 19 + TypeScript ~5.9 app at the exact STACK.md versions, vendor the `mzpeakts` reader via git submodule + `file:` install, and prove the full stack works in-browser end-to-end with ONE real user interaction.

The thin end-to-end slice: open a REAL demo `.mzpeak` from a URL -> read+display REAL file metadata + manifest -> select a spectrum index -> see its reconstructed m/z + intensity arrays plotted in uPlot -> served by `npm run dev` / `npm run preview`.

This plan also bootstraps the PROC-01 Codex review harness (`tools/codex_review.sh`) so this phase itself is reviewable, and records the architectural backbone in SKELETON.md.

Purpose: De-risk the single biggest "does it even run in the browser" unknown (vendored WASM reader, no COOP/COEP) before any other plan builds on it. Everything in Phase 1+ imports from the scaffold this plan creates.
Output: A runnable, deployable app shell with a working reader -> metadata -> single-spectrum-plot round trip, plus the review harness and SKELETON.md.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| URL/file -> WASM Parquet parser | Untrusted file bytes parsed by parquet-wasm/zip.js in the browser tab |
| npm/git-submodule supply chain | Third-party reader + parquet-wasm WASM binary pulled into the build |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Denial of Service | parquet-wasm parsing a malformed/huge .mzpeak | accept | Single-user static tool; tab-local blast radius only. Grid-dimension sanity caps land with imaging in P2; worker isolation in P5. |
| T-01-02 | Information Disclosure | accidental file upload via telemetry/CDN | mitigate | Fully client-side by design; no analytics/telemetry added; no network egress of file content (privacy is a product feature per PITFALLS security table). |
| T-01-SC | Tampering | npm installs + git submodule + vendored parquet-wasm .wasm | mitigate | Pin exact versions (STACK.md) + commit lockfile; pin submodule to a specific commit; parquet-wasm@0.7.1 is the package STACK.md audited as identical to the upstream vendored tgz. No `[ASSUMED]`/`[SUS]` packages introduced beyond the audited reader chain. |
</threat_model>
