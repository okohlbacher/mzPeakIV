Make per-spectrum signal reads route explicitly by `MS:1000525` representation (DATA-03 / IMAGING-SPEC C6). Add `getSpectrumArraysFor(reader, index, representation)` to `src/reader/arrays.ts` that selects the centroid source vs the data-array source by representation rather than the current incidental try-order, then rewire `store.selectSpectrum` to look up `spectrumMeta(...).representation` and call it. Purpose: DATA-03 demands deterministic, testable file routing — never "assume `spectra_data`". The store's single `selectSpectrum` action is shared by the numeric index input AND (in plan 03-03) the pixel-click, so routing lives in one place.
Output: a representation-routed reader variant + its mock-reader tests, and a rewired `selectSpectrum`.

Interpretation note (RESEARCH A2, confirm in Codex round1): D-07 is implemented as "call `getSpectrum`, then select the array source by representation", NOT exclusive per-file handle isolation. `getSpectrum` already loads whichever underlying file has rows; this variant makes the *choice of source array* deterministic per `MS:1000525`. Deeper file-level handle routing is a larger reader change deferred unless Codex/operator requires it.
